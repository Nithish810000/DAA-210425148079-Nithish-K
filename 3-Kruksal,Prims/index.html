02 / TWO STRATEGIES

Different journeys.
Same minimum.

Both are greedy algorithms. Kruskal thinks globally about the lightest remaining edge, while Prim expands locally from the tree it has already built.

01

EDGE-FIRST

Kruskal’s Algorithm

Sort edges by weight. Add the next lightest edge only when it connects two different components. Union–Find detects cycles efficiently.

Main structure
Union–Find
Time
O(E log E)
Strong fit
Sparse graphs
02

VERTEX-FIRST

Prim’s Algorithm

Start from one vertex. Repeatedly choose the cheapest edge that connects the growing tree to a new vertex. A min-heap keeps that choice fast.

Main structure
Min-heap
Time
O(E log V)
Strong fit
Dense graphs
